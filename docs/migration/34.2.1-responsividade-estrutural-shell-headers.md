## 1. BLOCO EXECUTADO

Bloco `34.2.1` executado como responsividade estrutural de shell, headers locais e conflitos de viewport.

Objetivo aplicado: impedir que shell, topbar, sidebar, gutters e headers locais prejudiquem leitura e uso das paginas em desktop, notebook e tablet, sem alterar backend, dados, handlers, rotas ou contratos.

## 2. MUDANCA APLICADA

Mudancas materiais:

- `frontend/src/shared/ui/tokens.css`
  - adiciona `--space-layout-content-fluid`;
  - adiciona `--space-layout-content-narrow`;
  - cria base de gutter fluido para desktop/notebook/tablet e gutter compacto para viewport estreito.

- `frontend/src/shared/ui/primitives.css`
  - reforca `min-width: 0` em primitives estruturais;
  - transforma `ui-page-header` em grid responsivo com `minmax(0, 1fr) auto`;
  - faz o header compartilhado colapsar para uma coluna em viewport menor;
  - evita que `page-header-actions` force largura minima estrutural.

- `frontend/src/app.css`
  - torna a topbar um grid estrutural com coluna de conteudo e coluna de contexto/acao;
  - aplica gutters fluidos ao `content` e a topbar;
  - centraliza e limita a largura util com `width: min(100%, var(--size-content-wide))`;
  - adiciona breakpoint intermediario em `1180px` para topbar e headers locais antes do mobile;
  - adiciona gutter mais compacto em `640px`;
  - adiciona guard final especifico para o header local do Dashboard, que tinha seletor mais forte e poderia vencer o breakpoint intermediario.

- `frontend/src/features/dashboard/page.js`
  - ajusta apenas a ordem das classes do header local para preservar o contrato visual compartilhado `priority-page-header ui-page-header ui-surface`;
  - sem mudanca de markup funcional, dados, links, handlers ou contrato.

- `tests/contract/test_frontend_responsive_shell_structure.py`
  - adiciona teste contratual para proteger gutters fluidos, colapso de header, breakpoint intermediario e contrato de shell preservado.

Validacao estrutural por breakpoint no CSS gerado:

| breakpoint | leitura validada |
| --- | --- |
| Desktop | topbar e headers mantem composicao em duas colunas com gutter fluido. |
| Notebook/intermediario (`<=1180px`) | topbar e headers compartilhados colapsam para uma coluna antes de esmagar titulo/acoes. |
| Tablet (`<=900px`) | shell colapsa para uma coluna, sidebar vira painel mobile e conteudo usa padding mobile. |
| Estreito (`<=640px`) | topbar e conteudo reduzem gutter lateral para preservar largura util. |
