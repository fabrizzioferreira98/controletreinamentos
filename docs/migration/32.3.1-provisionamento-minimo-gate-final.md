## 1. Bloco executado

| campo | valor |
| --- | --- |
| bloco | `32.3.1-provisionamento-minimo-gate-final` |
| objetivo | verificar e, se houver fonte real, provisionar as pre-condicoes obrigatorias do `run_release_strict.py` para eliminar o estado `BLOQUEADO` por variavel ausente. |
| candidato final | `release_20260420_2932_210420` em `homolog`. |
| gate oficial | `ops/scripts/release/run_release_strict.py` |

Fontes avaliadas e descartadas para provisionamento:

| fonte | classificacao | motivo para nao usar |
| --- | --- | --- |
| Ambiente atual do processo | fonte real aplicavel, mas vazia | Nenhuma das cinco variaveis esta definida. |
| `.env.example` | template | Contem valores de exemplo, placeholders ou campos vazios; usar isso violaria a regra de nao usar fake. |
| `ops\windows\env\hml.env.example` | template | Arquivo de exemplo; nao contem segredos reais de homolog. |

| `environment_parity_minimum.json` do pacote homolog | evidencia historica | Confirma que o runtime historico tinha `DATABASE_URL_configured=true`, mas nao fornece valor material para o gate local nem cobre as cinco pre-condicoes. |

Mudanca efetiva: nenhuma variavel foi provisionada, porque nao foi encontrada fonte real, aplicavel e segura para as cinco pre-condicoes. O bloqueio remanescente foi demonstrado por preflight.

## 3. Prova material

### 3.1 Fonte do requisito

O `run_release_strict.py` exige exatamente:

```text
DATABASE_URL
E2E_DATABASE_URL
E2E_LOGIN
E2E_PASSWORD
RELEASE_EVIDENCE_SIGNING_KEY
```

Essas variaveis sao verificadas antes de o gate interno prosseguir.

### 3.2 Verificacao do ambiente atual

Resultado da verificacao local:

| variavel | presente | tamanho |
| --- | --- | ---: |
| `DATABASE_URL` | `false` | `0` |
| `E2E_DATABASE_URL` | `false` | `0` |
| `E2E_LOGIN` | `false` | `0` |
| `E2E_PASSWORD` | `false` | `0` |
| `RELEASE_EVIDENCE_SIGNING_KEY` | `false` | `0` |

### 3.3 Preflight curto do gate

Comando executado:

```powershell
.\.venv\Scripts\python.exe ops\scripts\release\run_release_strict.py --base-url http://127.0.0.1:8102 --evidence-manifest C:\srv-data\controle-treinamentos\homolog\evidence\release_20260420_2932_210420\release_evidence_manifest.json --regression-checklist C:\srv-data\controle-treinamentos\homolog\evidence\release_20260420_2932_210420\regression_audit_checklist.md --evidence-max-age-hours 24
```

Resultado:

```text
exit code: 1
FAIL: variaveis obrigatorias ausentes: DATABASE_URL, E2E_DATABASE_URL, E2E_LOGIN, E2E_PASSWORD, RELEASE_EVIDENCE_SIGNING_KEY
```

Log estruturado relevante:

```json
{
  "event": "release_gate_blocked",
  "component": "run_release_strict",
  "release_id": "release_20260420_2932_210420",

```

Leitura: o gate nao deixou de estar bloqueado por variaveis obrigatorias ausentes. O resultado permanece `BLOQUEADO`, nao `FAIL` funcional do produto.
