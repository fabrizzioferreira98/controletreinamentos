# 44 - Frontend publish mirror guard

## Bloco executado

Hardening operacional do processo de publicacao do frontend estatico em Windows/self-hosted.

## Causa material

O frontend SPA usa imports ES module com cache busting por `?v=<timestamp>`. Se o diretorio produtivo receber apenas parte do build novo, ou se arquivos antigos ficarem orfaos no destino, o navegador pode carregar uma combinacao incoerente: entrada nova com modulos antigos, ou cache stale apontando para uma arvore nao espelhada.

O sintoma observado foi login aceito pelo backend (`auth_ok`), mas experiencia de login inconsistente no dominio publico por causa de frontend publicado de forma parcial/mista.

## Mudanca aplicada

Criado `ops/windows/scripts/Publish-Frontend.ps1`.

O script:

- gera build do frontend em diretorio temporario;
- injeta env de frontend quando `C:\srv\controle-treinamentos\env\prod.env` existir;
- valida arquivos obrigatorios (`index.html`, `app.js`, `app/bootstrap.js`, `config.js`, `app.css`);
- exige versao unica de assets em `*.html` e `*.js`;
- rejeita diretorios duplicados/orfaos conhecidos (`app/app`, `features/features`, `compat/compat`);
- cria backup do frontend atual;
- publica com `robocopy /MIR` para espelhar o build real e remover orfaos antigos;
- valida novamente a arvore publicada.

## Regra operacional

Publicacao manual por copia parcial nao deve ser usada para frontend SPA. A trilha segura e:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\ops\windows\scripts\Publish-Frontend.ps1
```

Para simulacao sem aplicar:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\ops\windows\scripts\Publish-Frontend.ps1 -WhatIf
```

## Regressao minima

- Nenhum contrato HTTP alterado.
- Nenhum endpoint alterado.
- Nenhuma permissao alterada.
- Nenhuma logica de autenticacao alterada.
- Mudanca restrita a operacao de build/publicacao do frontend.

## Criterio de fechamento

`fechado` quando novas publicacoes do frontend usarem espelhamento validado e deixarem de aceitar arvore produtiva com assets mistos ou orfaos.
