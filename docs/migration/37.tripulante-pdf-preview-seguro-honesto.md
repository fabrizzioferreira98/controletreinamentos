# 37.tripulante-pdf-preview-seguro-honesto

## Bloco Executado

Correcao segura e honesta do preview PDF da aba `Tripulantes`, sem alterar contrato HTTP, endpoints, permissoes ou backend de storage.

## Causa Material

O preview quebrava por combinacao de quatro causas:

- o frontend entregava a URL autenticada diretamente ao `iframe`;
- o endpoint de arquivo retornava PDF `inline`, mas tambem podia retornar erro JSON/404 quando o blob nao existia ou a sessao nao autorizava;
- o Caddy aplicava `X-Frame-Options: DENY` globalmente;
- a UI nao respeitava suficientemente `blob_available`, permitindo promessa visual de preview/download para metadata sem blob.

## Mudanca Aplicada

Frontend:

- `blob_available=false` passou a bloquear preview, abertura e download no painel de detalhe;
- o `iframe` deixou de receber a URL da API diretamente;
- o preview embutido passou a carregar o PDF por `fetch` autenticado, validar `response.ok`, validar `Content-Type: application/pdf` e so entao criar uma URL `blob:`;
- respostas JSON, `401`, `404` ou payload vazio passam a cair em fallback visual controlado;
- a lista exibe disponibilidade real por item, separando documento disponivel de metadata sem arquivo fisico.

Backend/edge:

- o endpoint de preview PDF passa a emitir `X-Frame-Options: SAMEORIGIN` e `Content-Security-Policy` com `frame-ancestors 'self'` somente para a acao de preview;
- o Caddy mantem `X-Frame-Options: DENY` global e aplica excecao escopada aos endpoints autenticados `/api/v1/tripulantes/*/files/*`.

## Teste / Check Focal

Checks contratuais cobrem:

- binario PDF autenticado preservado;
- `Content-Disposition: inline` para preview;
- `Content-Disposition: attachment` para download;
- headers de acesso autenticado preservados;
- excecao de embed escopada sem remover `DENY` global;
- frontend usando `blob_available`, `fetch`, `Content-Type` e `URL.createObjectURL`;
- fallback controlado para indisponibilidade fisica.

## Regressao Minima

- Nenhum endpoint novo.
- Nenhum contrato HTTP alterado.
- Nenhuma permissao removida.
- Nenhum bypass de autenticacao.
- Nenhum cleanup de legado.
- Nenhuma dependencia na reconciliacao completa dos PDFs antigos.

## Criterio de Fechamento

`fechado` quando:

- PDF disponivel gera preview real via blob autenticado;
- PDF indisponivel nao promete preview/download;
- JSON/404 bruto nao entra no iframe;
- `DENY` global permanece ativo fora da excecao escopada.

## Divida Rebaixada ou Remanescente

Rebaixada:

- preview falso por metadata sem blob;
- exposicao de erro bruto no painel visual;
- conflito direto entre `iframe` e `X-Frame-Options: DENY`.

Remanescente:

- recuperacao completa do legado `ids=30..41`, bloqueada por ACL no `Temp` legado;
- refinamento visual amplo da aba `Tripulantes`, fora do escopo desta etapa;
- correcao dos textos com encoding quebrado, fora do escopo desta etapa.
