# 32.outside.resume-kit - Kit minimo de retomada futura do gate operacional final

## 1. Bloco executado

| campo | valor |
| --- | --- |
| bloco | `32.outside.resume-kit-retomada-futura-gate-final` |
| natureza | kit operacional futuro, externo a Frente 32 ja encerrada |
| base principal | `32.outside.2-protocolo-minimo-retomada-gate-final` |
| encerramento externo | `32.outside.final.b-encerramento-executivo-externo-definitivo` |
| candidato de referencia | `release_20260420_2932_210420` |
| ambiente | `homolog` |
| reabre Frente 32 agora | nao |
| altera candidato/manifest/checklist/pacote | nao |
| emite `GO_OPERACIONAL_FINAL` | nao |
| cria novo requisito | nao |

Este kit nao executa o gate. Ele apenas consolida, em formato curto e reutilizavel, a sequencia minima que volta a fazer sentido se houver novo fato material externo.

## 2. Condicao de disparo

A retomada futura so pode ser iniciada por fato material externo, nao por decisao documental interna.

| gatilho | criterio obrigatorio |
| --- | --- |
| fonte viva real de homolog | as quatro variaveis existem em fonte operacional real, nao em template, backup, exemplo, historico, quarentena ou producao fora do escopo. |
| materialidade redigida | cada variavel pode ser comprovada com origem, `present=true` e `length>0`, sem expor valor. |
| carregamento possivel | as quatro variaveis podem ser carregadas no processo real do gate. |
| preflight destravavel | `_required_env()` do `run_release_strict.py` deve poder executar sem `missing_required_env`. |

Sem esse gatilho externo, o estado correto continua `BLOQUEADO DEFINITIVO DA FRENTE POR BLOQUEIO EXTERNO`.

## 3. Pre-condicoes materiais

| dependencia | pre-condicao material |
| --- | --- |
| `E2E_DATABASE_URL` | presente em fonte viva homolog, `present=true`, `length>0`, carregavel pelo processo do gate. |
| `E2E_LOGIN` | usuario E2E real de homolog, `present=true`, `length>0`, carregavel pelo processo do gate. |
| `E2E_PASSWORD` | segredo real correspondente, `present=true`, `length>0`, carregavel sem exposicao de valor. |
| `RELEASE_EVIDENCE_SIGNING_KEY` | chave HMAC real de homolog, `present=true`, `length>0`, carregavel pela trilha oficial de assinatura/validacao. |

Pre-condicoes transversais:

| item | regra |
| --- | --- |
| segredo | nunca imprimir valor, apenas origem, presenca e comprimento. |
| ambiente | usar `homolog`; nao usar producao fora do escopo. |
| candidato | manter `release_20260420_2932_210420`, salvo decisao externa futura formal de novo candidato. |
| documentacao | nao usar documento, workflow ou checklist como substituto de valor material no processo. |

## 4. Ordem minima de retomada

| ordem | passo | saida minima esperada |
| ---: | --- | --- |
| 1 | materializacao transitoria do env real | quatro variaveis carregadas no processo; preflight do `run_release_strict.py` sem `missing_required_env`; tabela redigida com origem, `present` e `length`. |
| 2 | assinatura HMAC real | assinatura gerada pela trilha oficial com `RELEASE_EVIDENCE_SIGNING_KEY`; validacao assinada sem `signature_missing` e sem `signing_key_env_missing`. |
| 3 | rerun completo do gate oficial | `run_release_strict.py` executado ate o fim contra o candidato e pacote aplicaveis, com saida material, exit code e classificacao objetiva. |
| 4 | ratificacao coerente de manifest/checklist/release_decision | reemitir somente o necessario para refletir gate, smoke, assinatura e decisao material; recalcular hashes se houver mudanca. |
| 5 | decisao futura correta | emitir exatamente uma decisao futura: `GO_OPERACIONAL_FINAL`, `NO_GO_OPERACIONAL_FINAL` ou `BLOQUEADO`, conforme resultado material. |

Esta ordem nao autoriza GO por presuncao. Ela apenas define o caminho minimo para uma decisao futura materialmente valida.

## 5. O que deve ser reexecutado

| item | quando reexecutar | motivo |
| --- | --- | --- |
| materializacao transitoria do env | sempre que o gatilho externo for comprovado. | provar que as quatro variaveis existem no processo real do gate. |
| assinatura HMAC real | apos `RELEASE_EVIDENCE_SIGNING_KEY` estar carregada. | eliminar `signature_missing` e `signing_key_env_missing`. |
| gate oficial completo | apos preflight sem `missing_required_env` e assinatura aplicavel. | obter resultado operacional material, nao documental. |
| ratificacao de manifest/checklist/release_decision | apos novo resultado material do gate/assinatura. | remover stale e alinhar pacote a decisao real. |
| decisao operacional futura | apenas apos pacote coerente e gate material classificado. | decidir GO/NO-GO/BLOQUEADO sem misturar governanca e aceite operacional. |

## 6. O que nao deve ser reaberto

| item | regra |
| --- | --- |
| abertura da Frente 32 | nao reabrir. |
| diagnostico curto | nao refazer. |
| matriz de blocos executaveis | nao refazer. |
| matriz de evidencias finais | nao refazer sem novo artefato material. |
| candidato congelado | nao trocar por conveniencia. |
| smoke final corrigido | nao rerodar por estetica; reavaliar apenas se a ordem operacional futura exigir apos gate destravado. |
| governanca/compat/topologia | nao usar como substituto do gate operacional final. |
| frentes 18, 27, 28, 29, 30 e 31 | nao reabrir. |

## 7. Resultados possiveis

| resultado futuro | criterio objetivo |
| --- | --- |
| `GO_OPERACIONAL_FINAL` | quatro variaveis materializadas, assinatura HMAC valida, gate oficial completo em PASS, pacote ratificado sem stale e `release_decision.md` coerente. |
| `NO_GO_OPERACIONAL_FINAL` | pre-condicoes satisfeitas, assinatura valida e gate oficial completo executado ate o fim com reprova funcional. |
| `BLOQUEADO` | qualquer pre-condicao material ausente, ambigua, nao carregavel ou assinatura obrigatoria impossibilitada. |

Ausencia material continua sendo `BLOQUEADO`, nao `NO_GO_OPERACIONAL_FINAL`.
