## Gap analysis entre dashboard atual e referencia

Incorporado:

- hierarquia superior por KPIs de vencimento;
- alerta de prioridade mais claro;
- fila critica como bloco principal de tratativa diaria;
- atalhos visuais para filtros ja existentes da lista completa;
- status com barras proporcionais;
- base operacional em cards mais densos;
- calendario com grade e detalhe mais limpos;
- agenda compacta separada como bloco proprio.

Adaptado:

- filtros da tabela viraram atalhos para a lista completa, sem esconder linhas localmente e sem criar contrato novo;
- calendario manteve `wireDashboardCalendar(calendarData)` e o payload real;
- a agenda continua usando `calendar.upcoming`, apenas reposicionada visualmente;
- tokens/primitives existentes foram reaproveitados, sem criar design system novo.

Ficou fora:

- qualquer mudanca na sidebar;
- navbar, shell global, loader, guard, permissao, SSR, deploy ou runtime;
- redesign de tripulantes, treinamentos, relatorios ou formularios;
- alteracao de API, contrato de dados ou ordenacao server-side da fila critica.
