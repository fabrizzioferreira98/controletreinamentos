# 35.tv-panel - Sincronia espacial cross-block

## Bloco executado

`Painel TV - sincronizacao visual e tatica entre mapa, Base em foco e Resumo das bases`.

Decisao do bloco: `fechado`.

## Gaps de sincronia identificados

- mapa, Base em foco e resumo lateral compartilhavam dados, mas nao um estado visual comum;
- a base ativa do mapa nao criava uma assinatura visual suficiente nos blocos laterais;
- o resumo indicava foco por classe, mas sem badge/editorial claro;
- severidade e halo visual nao eram propagados como narrativa unica;
- a leitura ainda parecia `mapa + card + lista`, nao camada espacial integrada.

## Validacao comparativa vs alvo

Recuperado:

- a base ativa do mapa agora propaga severidade para Base em foco e Resumo;
- o resumo lateral confirma visualmente a mesma base ativa;
- a camada espacial passa a ter uma narrativa unica: mapa indica, Base em foco explica, resumo confirma;
- cores, bordas, halos e badge convergem para a mesma prioridade operacional.

Ainda abaixo do alvo:

- validacao visual em TV real ainda pode pedir ajuste fino de intensidade do halo/conector;
- tile set cartografico custom continua fora desta etapa;
- captura automatizada em browser segue limitada pelo bloqueio local do runtime Node.

- nenhum contrato alterado;

- nenhuma rota nova;

## Criterio de fechamento

`fechado`: mapa, Base em foco e Resumo lateral passam a compartilhar estado visual e tatico, parecendo uma camada espacial unica.

## Divida rebaixada ou remanescente

Rebaixada:

- blocos corretos porem separados;
- foco ativo pouco propagado;
- resumo lateral com baixa confirmacao visual;
- severidade sem assinatura compartilhada;
- leitura espacial fragmentada.

Remanescente:

- calibracao visual final em TV real;
- tile set cartografico custom em etapa futura se a meta for paridade maxima com o alvo;
- browser/screenshot automatizado depende de liberar `node.exe` no ambiente.
