# 35.tv-panel - Leaflet premium broadcast map styling

## Bloco executado

`Painel TV - refinamento premium do mapa Leaflet real da Gestao de Bases`.

Decisao do bloco: `fechado`.

## Gaps visuais especificos do Leaflet atual

- tiles OSM ainda denunciavam mapa web padrao;
- filtro inicial escurecia o mapa, mas nao criava uma superficie nativa de command center;
- atribuicao e container mantinham cara de embed externo;
- marcadores pareciam cards sobrepostos ao mapa;
- faltava HUD operacional integrado ao mapa;
- faltava moldura cartografica com vinheta, grid e continuidade visual com fila/base em foco/ticker.

## Mudanca aplicada

Arquivos funcionais alterados:

- `backend/src/controle_treinamentos/static/tv-panel.js`;
- `backend/src/controle_treinamentos/static/tv-panel.css`.

Arquivos de registro:

- `docs/migration/35.tv-panel-leaflet-premium-broadcast-map-styling.md`;
- `docs/migration/README.md`;
- `README.md`.

Mudancas materiais:

- Leaflet continua sendo o stack do mapa;
- endpoint `bases.api_dados` e alternancia automatica foram preservados;
- adicionada HUD operacional do mapa com base ativa, meta de risco e contagem de bases;
- prefixo de atribuicao do Leaflet foi reduzido para linguagem operacional;
- tiles OSM receberam tratamento mais escuro, frio e integrado;
- shell do mapa recebeu vinheta, grid tecnico e moldura broadcast;
- marcadores foram refinados para parecerem pins/labels premium, nao cards web;
- estados ativos e criticos ganharam profundidade sem excesso de glow.

## Validacao comparativa vs alvo

Recuperado:

- mapa passa a parecer mais integrado ao painel dark premium;
- tiles perdem parte relevante da aparencia de OSM padrao;
- HUD aproxima o mapa da linguagem broadcast/command center;
- marcadores ficam menos parecidos com cards e mais com labels taticos;
- base ativa fica mais conectada ao mapa, resumo e Base em foco.

Ainda abaixo do alvo:

- continua usando tiles OSM filtrados, nao um tile set cartografico customizado;
- validacao visual em TV real ainda pode pedir ajuste fino de filtro/zoom;
- sem captura automatizada de screenshot no ambiente porque o runtime Node/browser esta bloqueado por permissao local.

## Regressao minima

- nenhum contrato alterado;
- nenhum endpoint novo;
- nenhuma rota nova;
- nenhum timer/polling principal alterado;
- fullscreen e permissoes preservados;
- alternancia automatica do mapa preservada;
- fallback para mapa anterior preservado.

## Criterio de fechamento

`fechado`: o mapa reduz claramente a cara de Leaflet adaptado e passa a operar como peca visual mais nativa do command center, preservando a fonte real da Gestao de Bases.

## Divida rebaixada ou remanescente

Rebaixada:

- tiles com cara de mapa web padrao;
- container com aparencia de embed;
- marcadores com cara de card sobreposto;
- ausencia de HUD operacional no mapa;
- baixa continuidade visual com o restante do painel.

Remanescente:

- tile set cartografico 100% custom/premium pode ser etapa futura;
- calibracao visual em TV real pode ajustar filtro, zoom e contraste;
- checagem automatizada via browser depende de liberar `node.exe`/runtime local.
