# 31.5.1 - Local build gerado

## 1. Bloco executado

Bloco: `31.5.1-local-build-gerado`.

Escopo:

- `.venv/` como excecao local de workspace;
- `frontend/dist/` como artefato local gerado.

## 2. Mudanca aplicada

Ratificacao documental. Nenhum arquivo de `.venv/` ou `frontend/dist/` foi editado, removido ou recriado.

Decisao:

| item | status | regra |
| --- | --- | --- |
| `.venv/` | excecao local permitida | pode existir apenas como ambiente local de desenvolvimento; nao e topologia oficial nem autoridade runtime. |
| `frontend/dist/` | build gerado local/publicavel | pode existir apenas como saida de `frontend/scripts/build_frontend.py`; nao e fonte viva nem criterio arquitetural. |

Build nao foi rerodado porque este bloco nao alterou `frontend/src/`, `frontend/scripts/` ou o conteudo de `frontend/dist/`.

## 3. Prova material

Existencia controlada:

```text
.venv=exists,is_dir=True
frontend\dist=exists,is_dir=True
frontend\src=exists,is_dir=True
frontend\scripts=exists,is_dir=True
venv_within_root=True
dist_within_root=True
```

Conteudo observado em `frontend/dist/`:

```text
app.css
app.js
config.js
index.html
lib.js
login-citation-jet.jpg
pages-dashboard-tripulantes.js
pages-training-workspace.js
pages-treinamentos-relatorios.js
shell.js
```

Coerencia documental:

- `docs/governance/repo-topology.md` declara `.venv/` e `frontend/dist/` como excecoes locais que nao definem a topologia oficial.
- `docs/operations/canonical-commands.md` declara `frontend/dist` como saida gerada/publicavel, nao fonte viva.
- `docs/architecture/FRONTEND_ARCHITECTURE.md` declara `frontend/src/` e `frontend/scripts/build_frontend.py` como fonte/build tooling, e `frontend/dist/` como saida gerada.
- `ops/scripts/repo/validate_repo_hygiene.py` possui excecoes explicitas para `.venv` e `frontend/dist` nos caminhos permitidos.

## 4. Teste / check

Comando executado:

```powershell
$env:PYTHONDONTWRITEBYTECODE='1'; .\.venv\Scripts\python.exe -B ops\scripts\repo\validate_repo_hygiene.py --root .
```

Resultado:

```text
success=true
violations=[]
```

Check adicional:

```text
.venv/ existe no root do workspace.
frontend/dist/ existe no caminho canonico de build.
frontend/dist/ nao foi editado manualmente neste bloco.
build frontend nao foi rerodado.
```

## 5. Criterio de fechamento

Fechado.

`.venv/` fica ratificado como excecao local, e `frontend/dist/` fica ratificado como artefato local gerado. Nenhum dos dois passa a ser topologia oficial, fonte viva ou criterio arquitetural.

## 6. Divida rebaixada ou remanescente

| item | status |
| --- | --- |
| `.venv/` | remanescente tolerado ate onboarding validado sem ambiente local embutido. |
| `frontend/dist/` | remanescente tolerado como saida gerada; remover fisicamente depende de fluxo externo de build/publicacao. |
| Build frontend | nao executado neste bloco; necessario apenas quando houver mudanca em `frontend/src/` ou `frontend/scripts/`. |
