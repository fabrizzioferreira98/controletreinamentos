## 2. Leitura do `lib.js` antes da extracao

Responsabilidades antes concentradas em `frontend/src/lib.js`:

| responsabilidade | conteudo antes |
| --- | --- |
| runtime config | `config` lido de `window.__FRONTEND_CONFIG__`. |
| estado transversal | `state.session`, `state.csrfToken`, `state.flash`, `state.frontendPerf`. |
| performance | `resetFrontendPerf`, `startFrontendPhase`, `finishFrontendPhase`. |
| flash | sessionStorage, normalizacao de tipo, `showFlash`, `consumeFlash`. |
| tracing | request id e correlation id. |
| CSRF | inclusao de `X-CSRFToken` em mutacoes. |
| API client | timeout, fetch, JSON/blob, headers, erro normalizado e side effects de auth. |
| sessao | `refreshSession()`. |
| utilitarios/UI helpers | escape, feedback markup, empty state, filtros, acessibilidade, busy action, confirm, formatadores e status classes. |

## 3. Mudanca aplicada

Novos modulos:

| modulo | responsabilidade |
| --- | --- |
| `frontend/src/services/api-client.js` | HTTP client, timeout, headers, JSON/blob, `credentials: include`, erro normalizado e efeitos de auth existentes. |
| `frontend/src/services/session-service.js` | `refreshSession()` e sincronizacao de `state.session` com CSRF retornado pela API. |
| `frontend/src/services/csrf-service.js` | leitura/escrita minima de token CSRF e aplicacao de header em mutacoes. |
| `frontend/src/services/trace-service.js` | `createTraceId()` e `clientCorrelationId()`. |
| `frontend/src/state/app-state.js` | `config`, `state` minimo e medicao de fases em `window.__FRONTEND_PERF__`. |
| `frontend/src/state/flash-state.js` | estado de flash, sessionStorage e normalizacao de tipo de flash. |

`frontend/src/lib.js` permanece como facade temporaria para preservar imports existentes de paginas, shell e app. Ele reexporta os owners novos e conserva apenas helpers ainda nao migrados para `shared/`:

| permaneceu em `lib.js` | motivo |
| --- | --- |
| escape/buildErrorMessage/formatadores | ainda sao helpers compartilhados; migracao para `shared/` fica para bloco futuro. |
| feedback markup, empty row, filtros e a11y helpers | ainda usados por paginas grandes; nao abrir F04/F05 neste bloco. |
| `withActionBusy` e `confirmAction` | helpers de UI existentes; sem redesign ou componentizacao agora. |
| `hashQuery`, `routePath`, `buildHashHref`, capability helpers | facade compativel para consumidores atuais ate proxima fatia arquitetural. |
