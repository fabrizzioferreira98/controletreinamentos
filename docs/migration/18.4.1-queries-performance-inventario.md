# 18.4.1 - Queries e Performance

Status: inventario de leituras concluido, sem tuning.

## 1. INVENTARIO DE SUPERFICIES

- dashboard operacional -> leitura analitica/operacional -> `api/http/dashboard/routes.py` chama `application.dashboard`, que usa `repositories.dashboard_cache.build_dashboard_context`, `build_dashboard_calendar` e `_fetch_dashboard_critical_rows`.
- painel TV de vencimentos -> leitura operacional pesada -> `blueprints/dashboard/routes.py` e `application.dashboard.get_tv_vencimentos_data` usam `_build_panel_tv_payload`.
- relatorio de habilitacoes -> leitura analitica -> `api/http/relatorios/routes.py::api_relatorios_habilitacoes` chama `application.relatorios.get_habilitacoes_report_data` e `build_habilitacoes_consolidadas_context`.
- relatorio de produtividade -> leitura analitica pesada -> `api/http/relatorios/routes.py::api_relatorios_produtividade` e export PDF usam `calculate_competencia_consolidada` e `fetch_produtividade_conferencias_map`.
- listagem de tripulantes -> leitura transacional comum -> `api/http/cadastros/routes.py::api_tripulantes_list` usa `count_tripulantes` e `fetch_tripulante_list_page`.
- listagem de treinamentos -> leitura transacional critica -> `api/http/cadastros/routes.py::api_treinamentos_list` usa `count_treinamentos`, `build_training_filters` e `fetch_training_page`.
- workspace treinamentos por tripulante -> leitura operacional/relatorial -> `routes_training_program.py::api_training_program_records_list` usa `fetch_training_program_record_list`.
- cadastro raiz de treinamentos -> leitura administrativa -> `routes_training_program.py` lista tipos, segmentos e horas por `repositories.training_program`.
- arquivos de tripulante e anexos -> leitura compat/documental -> `tripulante_files.fetch_tripulante_file_rows` une `tripulante_arquivos_pdf` e `treinamento_anexos_pdf`.
- foto de tripulante -> leitura compat historica -> `application.tripulante_media` le `foto_storage_ref`, `foto_base64` e `possui_foto`.
- auditoria administrativa -> leitura administrativa/export -> `blueprints/admin/routes.py::auditoria_list` pagina; `auditoria_export_pdf` exporta sem pagina.
- monitoramento do sistema -> leitura operacional pesada -> `monitoring/_monitoring_impl.py::collect_system_monitoring_snapshot` executa contagens, checks de integridade, backup e jobs.
- fila de jobs -> leitura operacional frequente -> `infra/jobs.py` consulta fila por status, stale locks, `FOR UPDATE SKIP LOCKED` e snapshots.
- notificacoes diarias -> leitura operacional pesada -> `infra/mailer.py::fetch_notification_training_rows` le treinamentos vencidos/a vencer e mapa de notificacoes ja enviadas.
- backup/restore drill -> leitura operacional de validacao -> `infra/backup.py` pode listar tabelas e fallback logico le ate `1_000_000` linhas por tabela; `backup_restore_drill.py` compara contagens criticas.
- scripts de dados -> leitura operacional/compat -> `ops/scripts/database/*.py` leem importacao, snapshots, midia legada, reconciliacao, cleanup e consistencia.

## 2. QUERIES RELEVANTES

- `build_habilitacoes_consolidadas_context` -> roda em relatorio/API/export de habilitacoes -> alta criticidade por join `tripulantes` + `treinamentos` + `tipos_treinamento` sem paginacao.
- `calculate_competencia_consolidada` -> roda em relatorio/API/PDF/painel TV de produtividade -> alta criticidade por ler tripulantes ativos, missoes, pernoites, adicionais, regras e parametros por competencia.
- `fetch_training_program_record_list` -> roda em workspace `treinamentos-tripulantes` -> alta criticidade por joins amplos, `GROUP BY`, `COUNT(a.id)` e ausencia de paginacao.
- `_build_panel_tv_payload` -> roda em painel TV de vencimentos -> alta criticidade por multiplas contagens filtradas, rankings por base/tripulante e listas limitadas.
- `build_dashboard_context` + `build_dashboard_calendar` -> roda no dashboard inicial -> alta criticidade por ser superficie frequente e agregar contagens, calendario e fila critica.
- `count_treinamentos` + `fetch_training_page` -> roda na listagem de treinamentos -> media/alta criticidade por repetir contagem e pagina com os mesmos filtros.
- `count_tripulantes` + `fetch_tripulante_list_page` -> roda na listagem de tripulantes -> media criticidade por filtros, paginacao e colunas de foto/compat.
- `fetch_tripulante_file_rows` -> roda em documentos de tripulante -> media/alta criticidade por `UNION ALL` entre arquivos File e anexos de treinamentos.
- `auditoria_list` / `auditoria_export_pdf` -> roda em admin e export PDF -> media/alta criticidade; listagem pagina, export nao pagina.
- `fetch_notification_training_rows` -> roda em job/notificacao diaria -> alta criticidade por varrer vencimentos ate horizonte configurado e juntar tripulante/equipamento/tipo.
- `collect_system_monitoring_snapshot` -> roda em monitoramento admin -> alta criticidade por concentrar varias contagens e checks de integridade em uma tela.
- `sync_tripulantes_snapshot.py` -> roda em sincronizacao operacional -> alta criticidade por snapshot integral de bases, tripulantes e pilotos.
- `migrate_tripulante_media_to_storage.py` / `reconcile_tripulante_photos.py` -> roda em compat de midia -> alta criticidade por ler blobs/base64/storage refs historicos.
- `run_db_consistency.py --repair` -> roda em consistencia operacional -> alta criticidade por combinar leitura de checks com reparo estrutural.

## 3. LEITURA POR CATEGORIA

- transacional | `api_tripulantes_list`, `api_treinamentos_list`, detalhes de tripulante/treinamento, options de formularios, usuarios/notificacoes admin.
- analitica | relatorio de habilitacoes, relatorio de produtividade, auditoria export PDF, dashboards agregados.
- operacional pesada | painel TV, monitoramento, jobs worker, notificacoes diarias, backup/restore drill, scripts de sync/import/cleanup/consistencia.
- compat historica | `foto_base64`, `foto_storage_ref`, `possui_foto`, `storage_ref`, `arquivo_pdf`, `tripulante_arquivos_pdf` + `treinamento_anexos_pdf`, snapshots de tripulantes/pilotos.

## 4. HOTSPOTS INICIAIS

- relatorio de produtividade -> alto -> concentra leituras por competencia e recalculo em memoria sobre tripulantes, missoes, pernoites e adicionais.
- relatorio de habilitacoes -> alto -> materializa matriz tripulante x habilitacoes sem paginacao.
- workspace treinamentos por tripulante -> alto -> join amplo com segmentos, horas de voo, anexos e `GROUP BY`.
- painel TV/dashboard -> alto -> superficie frequente, cacheada, mas com varias contagens e rankings por janela de vencimento.
- notificacoes diarias -> alto -> job recorrente que le todos os treinamentos vencidos/a vencer no horizonte.
- monitoramento admin -> medio/alto -> tela unica agrega muitos checks e contagens operacionais.
- auditoria export PDF -> medio/alto -> exporta linhas filtradas sem `LIMIT/OFFSET`.
- compat de midia/fotos -> medio/alto -> depende de colunas e blobs historicos (`foto_base64`, `arquivo_pdf`, `storage_ref`) que podem carregar dados grandes.
- backup logico fallback -> alto -> le tabelas inteiras com limite alto quando `pg_dump` nao esta disponivel.

## 5. CONCLUSAO DA ETAPA 18.4.1

- O custo real do banco parece concentrado em leituras agregadas e sem paginacao: produtividade, habilitacoes, treinamentos por tripulante, painel/dashboard, notificacoes, monitoramento e exports. As listagens transacionais ja aparecem mais controladas por `COUNT + LIMIT/OFFSET`, mas ainda merecem medicao por frequencia. Compat historica e scripts operacionais ficam como segunda frente de custo por volume eventual e leitura de blobs/snapshots.
