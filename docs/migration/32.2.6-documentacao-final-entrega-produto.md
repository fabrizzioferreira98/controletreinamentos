# 32.2.6 - Documentacao final de entrega do produto

## 1. Bloco executado

| campo | valor |
| --- | --- |
| bloco | `32.2.6` |
| objetivo | consolidar a documentacao final de entrega do produto de forma objetiva, utilizavel e coerente com a trilha operacional real. |
| natureza | consolidacao documental operacional; nao e decisao final e nao e PASS por documentacao. |
| base | `32.0`, `32.1`, `32.2.0`, `32.2.1`, `docs/operations/RELEASE_EXECUTION_CHECKLIST.md` e `docs/operations/RELEASE_MANAGEMENT.md`. |
| decisao | documentacao viva consolidada; `GO_OPERACIONAL_FINAL` ainda nao emitido. |

Este bloco nao executa gate, nao corrige baseline vermelho, nao altera pacote externo de evidencias, nao executa deploy e nao reabre frentes encerradas.

## 2. Mudanca aplicada

Foram consolidados os documentos vivos ja existentes, sem criar documento operacional redundante:

| arquivo vivo | mudanca |
| --- | --- |
| `docs/operations/RELEASE_EXECUTION_CHECKLIST.md` | Adicionada a secao `Aceite final da entrega do produto`, com checklist final, ponteiros obrigatorios para evidencias e modelo minimo de `<evidence_root>/release/release_decision.md`. |
| `docs/operations/RELEASE_MANAGEMENT.md` | Adicionada a secao `Aceite final de entrega do produto`, separando `GO_DE_GOVERNANCA` de `GO_OPERACIONAL_FINAL` e instruindo o operador final. |
| `docs/migration/README.md` | Indexacao deste bloco da Frente 32. |

Nao foi criado novo template vivo separado porque `RELEASE_EXECUTION_CHECKLIST.md` ja e o checklist oficial de execucao de release e ja aponta para `release_decision.md` no pacote externo.

## 3. Prova material

Documentos finais vivos consolidados:

- `docs/operations/RELEASE_EXECUTION_CHECKLIST.md`
- `docs/operations/RELEASE_MANAGEMENT.md`

Ponteiros obrigatorios definidos para o pacote externo de evidencias:

| evidencia | ponteiro operacional |
| --- | --- |
| Checklist de release preenchido | `<evidence_root>/release/release_execution_checklist.md` |
| Checklist de rollback | `<evidence_root>/rollback/rollback_checklist.md` |
| Gate final strict | `<evidence_root>/gate/run_release_strict.log` |
| Smoke pos-release | `<evidence_root>/smoke/post_release_smoke.json` |
| Manifest de evidencias | `<evidence_root>/manifest/evidence-manifest.json` |
| Decisao final do operador | `<evidence_root>/release/release_decision.md` |

Modelo minimo registrado para `release_decision.md`:

- `release_id`;
- ambiente alvo;
- commit/identificador operacional;
- operador responsavel;
- data/hora da decisao;
- resultado entre `GO_OPERACIONAL_FINAL`, `NO-GO` e `BLOQUEADO`;
- provas materiais;
- justificativa;
- pendencias, desvios e riscos residuais.

## 4. Teste / check

Check documental executado:

| check | resultado |
| --- | --- |
| Checklist final de entrega existe em doc vivo | `RELEASE_EXECUTION_CHECKLIST.md` contem `Aceite final da entrega do produto`. |
| Documento de aceite final tem modelo minimo | `RELEASE_EXECUTION_CHECKLIST.md` define `<evidence_root>/release/release_decision.md`. |
| Ponteiro para evidencias obrigatorias existe | `RELEASE_EXECUTION_CHECKLIST.md` lista manifest, checklist, rollback, gate strict, smoke e decisao final. |
| Separacao entre governanca e entrega existe | `RELEASE_MANAGEMENT.md` declara que `GO_DE_GOVERNANCA` nao autoriza entrega de produto. |
| Instrucao ao operador existe | `RELEASE_MANAGEMENT.md` define quando emitir `GO_OPERACIONAL_FINAL`, `NO-GO` ou `BLOQUEADO`. |

Checks nao executados neste bloco:

- `run_release_strict.py`;
- `post_deploy_smoke.py`;
- validacao de manifest;
- validacao de checklist preenchido.

Motivo: este bloco consolida documentacao final. Os gates operacionais permanecem nos blocos proprios da Frente 32.

## 5. Criterio de fechamento

| criterio | status |
| --- | --- |
| Checklist final de entrega consolidado | fechado |
| Documento de aceite final definido sem criar template redundante | fechado |
| Ponteiros para evidencias obrigatorias definidos | fechado |
| `GO_DE_GOVERNANCA` separado de `GO_OPERACIONAL_FINAL` | fechado |
| Instrucao clara para operador final registrada | fechado |
| Documentacao nao convertida em PASS por si so | fechado |
| README de migracao indexado | fechado apos atualizacao |

Decisao do bloco: `fechado como consolidacao documental`.

Fechado nao significa `GO_OPERACIONAL_FINAL`, nao significa deploy autorizado e nao substitui evidencia material.

## 6. Divida rebaixada ou remanescente

| item | status | encaminhamento |
| --- | --- | --- |
| Candidato final 32.x | remanescente bloqueante | Precisa ser identificado em bloco proprio antes do aceite final. |
| Gate strict final | remanescente bloqueante | Precisa retornar `PASS` sem bypass para permitir `GO_OPERACIONAL_FINAL`. |
| Smoke final | remanescente bloqueante quando aplicavel | Precisa retornar `PASS` contra o ambiente final com metricas validas. |
| Manifest/checklist finais preenchidos | remanescente bloqueante | Precisam existir no pacote externo e apontar para o mesmo candidato. |
| `release_decision.md` preenchido | remanescente bloqueante | Deve ser gerado no pacote externo pelo operador final, com prova material. |
| Baselines vermelhos da 32.2.1 | remanescentes | Nao foram corrigidos neste bloco. |
