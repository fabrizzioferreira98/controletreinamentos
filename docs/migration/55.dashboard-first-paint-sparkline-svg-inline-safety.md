# 55 - Dashboard first paint sparkline SVG inline safety

## Bloco executado

Hardening localizado de first paint para sparkline SVG da dashboard.

## Componentes criticos identificados

- `frontend/src/features/dashboard/page.js`:
  - `renderDashboardSparkline(tone)` era o ponto critico.
  - O `<path>` do sparkline entrava sem invariantes inline (`fill/stroke`) e dependia de CSS para estado visual minimo correto.
- Demais SVGs inline mapeados na SPA (`shell/navigation`, `shell/render-shell`, icones da propria dashboard) ja possuem `fill="none"` e `stroke` inline.

## Mudanca aplicada

No bloco `renderDashboardSparkline`:

- `<svg ... fill="none">`
- `<path ... fill="none" stroke="currentColor" stroke-width="2.8" stroke-linecap="round" stroke-linejoin="round" vector-effect="non-scaling-stroke" />`

Com isso, a geometria minima do sparkline fica valida no first paint sem depender de CSS tardio.
