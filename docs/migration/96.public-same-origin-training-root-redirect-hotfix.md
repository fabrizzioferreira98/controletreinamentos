# 96. Public same-origin training root redirect hotfix

## Bloco executado

Hotfix de runtime para impedir que a URL publica `/tipos-treinamento` renderize a superficie SSR legada quando o frontend esta servido no mesmo dominio, mas `FRONTEND_PUBLIC_ORIGIN`/`FRONTEND_LOCAL_ORIGIN` nao estao configurados.

## Causa material

`/tipos-treinamento` ja redirecionava para `/#/treinamentos/raiz` quando `frontend_compat_enabled()` estava ativo.

Porem `frontend_compat_enabled()` dependia de `frontend_official_enabled()`, que so considerava variaveis `FRONTEND_PUBLIC_ORIGIN` ou `FRONTEND_LOCAL_ORIGIN`. Em producao same-origin, como `https://controle.brasilvida.app.br`, a origem valida tambem pode vir da propria requisicao via proxy. Sem essa variavel, a rota voltava a renderizar `tipos_list.html`.

## Mudanca aplicada

- `backend/src/controle_treinamentos/core/frontend_routes.py` agora reconhece requisicoes publicas same-origin como frontend oficial quando:
  - existe `request.origin`/host;
  - o host nao e `localhost`;
  - a requisicao nao e acesso direto ao backend local nas portas `8101`/`8102`.
- `frontend_redirect_origin()` ja preservava a origem da requisicao publica; a mudanca torna `frontend_compat_enabled()` coerente com essa politica.

## Teste focal

- `tests/contract/test_frontend_compat_redirects.py` cobre `https://controle.brasilvida.app.br/tipos-treinamento` autenticado, sem `FRONTEND_PUBLIC_ORIGIN`, e exige redirect para `https://controle.brasilvida.app.br/#/treinamentos/raiz`.
- O teste SSR backend-only em `localhost` continua preservado para nao quebrar compat local/legada.

## Criterio de fechamento

Fechado quando uma requisicao publica same-origin autenticada para `/tipos-treinamento` nao renderiza `tipos_list.html` e redireciona para `/#/treinamentos/raiz`, mesmo sem variaveis `FRONTEND_*_ORIGIN`.
