# 88. Remocao estrutural de Missoes

Data: 2026-04-29

## Bloco executado

Remocao de Missoes como feature viva, cobrindo frontend, backend, contratos, permissoes, templates SSR, bootstrap de schema e documentacao ativa.

## Mudanca aplicada

- Removidos links/CTAs de Missoes da navegacao SPA, dashboard SPA e dashboard SSR.
- Removidas rotas SSR `/missoes`, `/missoes/novo`, `/missoes/<id>/editar` e `/missoes/<id>/excluir`.
- Removidos use cases, repository methods, write model, form builders/options e templates exclusivos de Missoes.
- Pernoites foram desacoplados de `missao_id` no fluxo vivo e permanecem como compat SSR standalone.
- Removidos `missoes:*` de grupos de permissao, defaults, endpoint map, landing candidates e dependencias.
- Removido bootstrap de `missoes_operacionais`, `missao_tripulantes`, indices e columns map correspondentes.

## Politica de dados

Nao foi executado DROP fisico em banco vivo nesta etapa. Tabelas ou colunas historicas existentes devem ser tratadas por migracao destrutiva explicita, com backup, rollback e decisao de retencao.
