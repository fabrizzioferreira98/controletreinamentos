# 29.3.3 - Evidencia real de carga autenticada `20w` e `30w`

## Objetivo do bloco

Executar `load_authenticated_20w` e `load_authenticated_30w` contra o runtime real de homolog, gerar evidencia material dos dois cenarios e integrar os artefatos ao pacote oficial de release sem substituir homolog por carga local ou medicao ornamental.

## Mudancas aplicadas

- `ops/scripts/perf/load_test_authenticated.py`
  - manteve o login HTML como trilha primaria;
  - adicionou fallback curto para a trilha canonica de API (`/api/v1/session` -> `/api/v1/session/login`) quando a pagina HTML nao expoe CSRF;
  - preservou a validacao de preflight autenticado, falhas 401/403, erros de transporte, disponibilidade e p95.
- `ops/scripts/perf/run_authenticated_matrix.py`
  - corrigiu a consolidacao de valores zero (`percent_5xx=0`, falhas zero) para nao transformar ausencia de erro em falso negativo;
  - preservou os criterios estritos: disponibilidade >= 99%, p95 <= 1200 ms, 5xx <= 0.5%, sem falha de auth/permissao/login/preflight.
- `ops/scripts/release/run_authenticated_release_matrix.py`
  - passou a gerar artefatos canonicos por cenario (`load_auth_20w.json`, `load_auth_30w.json`) alem do agregado `load_auth_matrix.json`;
  - atualiza `checks.load_authenticated_20w` e `checks.load_authenticated_30w` no manifest;
  - atualiza o anexo `Carga autenticada` no checklist do pacote.
- `ops/scripts/release/release_gate.py`
  - quando a matriz autenticada e exigida com manifest, envia a saida para `perf/` dentro do proprio pacote de release e usa o wrapper de integracao.
- `tests/ops/test_load_test_authenticated_entrypoints.py`
  - cobre o fallback de login via API e a preservacao de `percent_5xx=0`.
- `tests/ops/test_release_e2e_homolog_entrypoints.py`
  - cobre a integracao do wrapper de carga com manifest/checklist e o encaixe pelo gate.
