## 2. Leitura do `app.js` antes da extracao

Responsabilidades antes concentradas em `frontend/src/app.js`:

| responsabilidade | conteudo antes |
| --- | --- |
| bootstrap | `boot()`, listener de `hashchange` e chamada inicial. |
| registro de rotas | objeto `routes` com hash routes estaticas. |
| loaders lazy | `routeModuleLoaders`, cache de modulo e `import()` das paginas vivas. |
| rotas dinamicas | `resolveDynamicRoute()` para `#/tripulantes/<id>` e `#/treinamentos/<id>`. |
| guards | `routeAllowed()` e render de acesso negado. |
| erro global | listeners `error` e `unhandledrejection`. |
| erro de sessao | indisponibilidade de backend/session e fallback de retry. |
| erro de rota | render de falha navegavel com retry. |
| dependencias criticas | `lib.js` para sessao/API/perf/state e `shell.js` para login/shell. |

## 3. Mudanca aplicada

`frontend/src/app.js` ficou como entrypoint fino:

```js
import { startApp } from "./app/bootstrap.js?v=20260404-1215";

void startApp();
```

Novos modulos:

| modulo | responsabilidade |
| --- | --- |
| `frontend/src/app/bootstrap.js` | ciclo de inicializacao: registra handlers uma vez, valida sessao, resolve hash atual, chama login, fallback, guard e render da rota. |
| `frontend/src/app/router.js` | runtime do router: `withFrontendPhase`, cache de modulos, `route_import`, `renderLazyRoute`, rotas estaticas materializadas e resolucao dinamica. |
| `frontend/src/app/route-registry.js` | fonte oficial de loaders, hash routes estaticas, rotas dinamicas, exports e permissoes. |
| `frontend/src/app/guards.js` | permissao por capability e render de acesso negado. |
| `frontend/src/app/errors.js` | erro global, indisponibilidade de sessao, tela de sessao indisponivel e falha de rota com retry. |

O que permaneceu em `app.js`:

| item | motivo |
| --- | --- |
| import de `startApp` | manter o contrato de entrypoint do `index.html`. |
| `void startApp()` | preservar inicializacao automatica da SPA. |
