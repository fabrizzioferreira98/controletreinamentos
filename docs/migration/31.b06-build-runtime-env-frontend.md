## 5. Mudanca aplicada

Foi alterado `docs/architecture/FRONTEND_ARCHITECTURE.md` para registrar:

- build nao e runtime;
- runtime nao e fonte viva;
- `.env.example` e template, nao prova de env real;
- trilha real de build;
- trilha real de runtime;
- classificacao de env/config, artefato, publicacao e acoplamentos backend/frontend;
- acoplamento explicito com `backend/src/controle_treinamentos/static/styles.css`;
- referencias `/static/*` como acoplamento backend/frontend;
- `Caddyfile.example` como prova de proxy backend, nao de publicacao estatica do frontend sozinho.

Foi alterado `frontend/README.md` para trocar a nocao de app estatico independente por app estatico separado, mas nao autonomo em runtime.

Foi adicionado `tests/contract/test_frontend_build_runtime_env_boundaries.py` para validar:

- `build_frontend.py` preserva `--output-dir`, `--env-file`, `config.js` e composicao CSS com backend;
- build com env customizado gera `config.js` em diretorio temporario fora do checkout;
- templates de env contem as chaves esperadas, sem serem tratados como env real;
- backend continua dono de origins/CORS/compat redirects;
- docs vivas registram as classificacoes e o acoplamento.
