# 81. Jornadas: read model de auditoria operacional

## Bloco executado

Rebaixamento da aba `#/relatorios/produtividade/jornadas` de frente principal de escrita para superficie de leitura, auditoria, calculo derivado e memoria operacional.

## Superficie mapeada

- Rota SPA: `#/relatorios/produtividade/jornadas`.
- UI principal: `frontend/src/features/relatorios/produtividade-jornadas-page.js`.
- Leitura backend preservada: `GET /api/v1/relatorios/produtividade/jornadas`.
- Exportacao preservada: `/produtividade/jornadas/export.pdf`.
- Origem operacional para ajuste: `/missoes`.

## Mudanca aplicada

- A tela deixou de exibir comandos de `Adicionar linha`, `Salvar` e `Remover`.
- O JavaScript da superficie deixou de chamar `POST`, `PUT`, `DELETE` e `preview` de Jornadas.
- A grade passou a renderizar campos brutos como valores somente leitura.
- Cada linha passou a apontar para a origem operacional em `Operacoes > Missoes`.
- A microcopy foi ajustada para explicitar que a tela le fatos operacionais e snapshot derivado.

## Fronteira preservada

- Endpoints legados de mutacao nao foram removidos nesta etapa, para preservar compatibilidade.
- Jornada continua calculada pelo backend e exibida como memoria/auditoria.
- Produtividade nao foi alterada e nao recebe `jornada_pagamento` como parcela automatica.
- Snapshot derivado continua sendo leitura de apoio, nao fato canonico.
- Missoes permanece a superficie correta para ajuste operacional.
