# 98. Operacoes/Pernoites Read API Boundary

## Bloco

`98.operacoes-pernoites-read-api-boundary`

## Contexto

Operacoes/Pernoites estava ratificado como SSR canonico atual, com API futura declarada em contrato, mas sem rota registrada. Esse estado era seguro para o fluxo vivo, porem deixava ambiguidade material entre "candidato futuro" e "contrato canonico".

## Decisao

Iniciar a migracao canonica de forma controlada:

- leitura de Pernoites passa a ter API real registrada;
- escrita continua SSR canonica atual neste ciclo;
- UI SSR viva nao e removida nem reapontada por big bang;
- POST/PUT/PATCH/DELETE API permanecem explicitamente nao registrados ate bloco proprio de cutover SPA/API.

## Superficie atual

- UI/listagem viva: `/pernoites`
- criacao viva: `/pernoites/novo`
- edicao viva: `/pernoites/<id>/editar`
- exclusao viva: `/pernoites/<id>/excluir`
- API leitura canonica registrada: `/api/v1/operacoes/pernoites`
- API detalhe leitura canonica registrada: `/api/v1/operacoes/pernoites/<id>`

## Contrato

`backend/src/controle_treinamentos/contracts/operacoes.py` agora classifica:

- `OPERACOES_SSR_CURRENT_ENDPOINTS` como UI/escrita SSR atual;
- `OPERACOES_READ_API_ENDPOINTS` como `api_read_canonical_registered`;
- `OPERACOES_FUTURE_API_CONTRACT.status` como `read_api_registered_write_ssr_canonical_current`.

## Guardas

- `tests/contract/test_operacoes_contract.py` prova registro real das rotas GET e ausencia de escrita API.
- `tests/architecture/test_api_route_repository_boundaries.py` impede que handlers API de leitura acessem repository/db diretamente.
- `tests/unit/test_endpoint_permission_coverage.py` cobre permissao dos endpoints registrados no mapa de autorizacao.

## Regra de saida

A proxima fase so pode migrar UI/escrita para SPA/API se:

- mantiver leitura API como contrato principal;
- registrar explicitamente rotas de escrita antes de qualquer consumo SPA;
- preservar fluxo SSR vivo ate prova de cutover;
- atualizar permissao, testes e documentacao no mesmo bloco.

## Criterio

`fechado`: a frente nao esta mais indefinida. Ha contrato API real de leitura, e a escrita SSR remanescente esta classificada como decisao operacional temporaria, nao como ambiguidade.
