# 76 - Jornadas: snapshot persistido do calculo

## Contexto

A leitura da aba `Lancamentos de Jornada` podia exibir/calcular linhas usando fallback de preview quando o backend nao encontrava resultado persistido:

`Resultado persistido de calculo nao encontrado; calculo preview aplicado.`

Esse fallback mantinha o valor calculado disponivel, mas enfraquecia a rastreabilidade entre o que foi salvo, reaberto e exportado.

## Mudanca

- Adicionadas colunas em `missoes_operacionais` para snapshot de calculo:
  - `calculo_jornada_json`
  - `calculation_version`
  - `calculation_run_id`
- Ao criar ou editar uma linha de jornada, o backend recalcula o recorte da competencia/tripulante e persiste o snapshot calculado.
- A leitura passa a usar `calculation_source=persisted` quando existe snapshot.
- Snapshots ausentes sao backfilled de forma nao destrutiva pela migration corretiva, sem sobrescrever calculos ja persistidos.
- Se uma linha ficar incompleta/invalida em edicao, snapshot antigo e limpo para evitar resultado stale.
- O aviso tecnico de fallback para preview agora so aparece quando existe preview calculado com sucesso; linhas invalidas exibem apenas os erros operacionais de preenchimento.
