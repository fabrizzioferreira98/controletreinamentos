# 100. Financeiro Horas Totais Voadas Backend Contract

## Bloco

`100.financeiro-horas-totais-voadas-backend-contract`

## Contexto

O gate HML da frente `Relatorio geral de horas` identificou `404` nos endpoints canonicos:

- `GET /api/v1/financeiro/horas-totais-voadas`
- `GET /api/v1/financeiro/horas-totais-voadas.pdf`

O contrato JSON ja existia no codigo-fonte local, mas a publicacao HML estava incompleta e o PDF canonico ainda nao estava registrado como rota/contrato/export real.

## Decisao

Publicar o contrato backend completo da frente:

- JSON consolidado mensal por competencia e funcao operacional;
- PDF canonico equivalente;
- permissionamento explicito;
- guardas de contrato HTTP e url_map;
- bloqueio de exportacao quando houver pendencia de calculo persistido.

## Fonte de verdade

O relatorio usa exclusivamente `financeiro_calculos_horarios` persistidos vigentes, via read model de Jornadas, filtrando:

- competencia;
- funcao operacional;
- `org_id`;
- linhas/missoes ativas;
- calculos nao obsoletos, nao cancelados e nao excluidos;
- sem preview.

## Migration de banco

Nao ha migration SQL nesta correcao. A mudanca e de contrato HTTP, use case de exportacao PDF, RBAC e documentacao. O schema financeiro vigente ja contem a fonte persistida necessaria.

## Guardas

- `FINANCE_HTTP_CONTRACTS` registra JSON e PDF.
- `ENDPOINT_PERMISSION_MAP` registra as rotas Flask reais.
- Testes de contrato comparam matriz declarada e `url_map` runtime.
- Teste unitario valida PDF com assinatura `%PDF` e marcador final `%%EOF`.
- Teste unitario bloqueia PDF quando existe lancamento ativo sem calculo horario persistido.

## Criterio

`fechado` somente quando HML responder JSON e PDF sem `404`, com `Content-Type: application/pdf` no PDF autenticado e `401` JSON controlado sem sessao.
